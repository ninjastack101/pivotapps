Permissions Allow access to current {{ skill }}

Permissions Are Inherited Only From Sub-Categories:

  • Please note that when creating a new category or sub-category, you will need to explicitly define permissions at every level for the company you are granting access
  • Permissions are not inherited by sub-categories from categories, or by categories from departments.
  • Permissions are inherited by skills from the sub-category.

How to Assign Permissions for This Skill:

  1. Assign permissions for a company to the department, category, and sub-category. Use the vertical ellipses button and choose Manage Settings to confirm the company has been associated at all three levels selected above: Department, Category, and Sub-Category. Company permission must be assigned at all three levels to allow users to access this skill.
  2. Associate users with the company. From the Companies tab, expand the Permissions panel, and select the Users tab. Confirm that intended users of this skill have been associated with the company.
  3. Associate users with the department. From the Companies tab, expand the Permissions panel, and select the User Departments tab. Confirm that intended users of this skill have been associated with the selected department.
